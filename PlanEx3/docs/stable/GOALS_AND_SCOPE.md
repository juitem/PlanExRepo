# GOALS_AND_SCOPE — 목표 / 범위 / 비범위

> **Knowledge 유형**: stable
> **갱신 조건**: 요구사항 범위 변경 시 (ADR 필수)

---

## 목표

### 주요 목표

1. **폴더 비교**: 두 폴더를 선택하여 파일/디렉토리 구조 차이를 시각적으로 확인
2. **폴더 병합**: 한 폴더에서 다른 폴더로 파일을 선택적으로 병합
3. **파일 비교**: 두 파일의 내용을 줄 단위로 diff 비교
4. **파일 병합**: 두 파일의 변경 내용을 병합하고 충돌을 해결
5. **VS Code 통합**: VS Code의 Native UI 패턴에 자연스럽게 통합

### 보조 목표

- AI 에이전트 오케스트레이션으로 개발 가능한 구조 확립
- 재진입 가능한 문서 기반 개발 운영체계 검증

## 범위 (In Scope)

### 핵심 기능

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 폴더 선택 | 두 개의 로컬 폴더를 선택하는 UI | P0 |
| 폴더 트리 비교 | 파일/디렉토리 구조를 side-by-side 트리로 표시 | P0 |
| 파일 상태 표시 | ADDED / DELETED / MODIFIED / SAME 상태 시각화 | P0 |
| 파일 복사 | 한쪽에서 다른 쪽으로 파일 복사 (단방향) | P0 |
| 선택적 병합 | 여러 파일을 선택해 일괄 복사/병합 | P1 |
| 파일 diff 뷰 | 텍스트 파일 내용을 줄 단위로 비교 | P1 |
| 파일 병합 | 두 파일 내용 병합 (충돌 해결 UI 포함) | P1 |
| 필터링 | 파일 확장자, 상태 기준으로 필터링 | P2 |
| 무시 패턴 | .gitignore 스타일의 무시 패턴 지원 | P2 |

### 지원 플랫폼

- VS Code 1.80+ (기준 버전 확정은 JOB-001에서)
- macOS, Windows, Linux (크로스 플랫폼)

## 비범위 (Out of Scope)

| 항목 | 이유 |
|------|------|
| 3-way 병합 (Git merge) | 복잡도 높음, 별도 도구 존재 |
| 원격 폴더 비교 (SSH, FTP) | 초기 버전 범위 초과 |
| 바이너리 파일 내용 비교 | 기술적 복잡도, 실용성 낮음 |
| 버전 관리 시스템 통합 | Git extension 이 대체 |
| 실시간 파일 감시 | 리소스 소비 우려 |
| 대용량 파일(100MB+) 지원 | 성능 리스크 |
| 클라우드 스토리지 연동 | 초기 버전 범위 초과 |

## 제약 조건

### 기술적 제약

- VS Code Extension API 범위 내에서만 동작
- Node.js 런타임 (VS Code 내장) 사용
- 파일 시스템 접근은 VS Code 워크스페이스 신뢰 모델 준수

### 운영 제약

- 모든 구현은 문서 기반 개발 프로세스 준수
- 코드 변경 전 ADR 생성 (아키텍처 결정 시)
- 각 Job/Phase 완료 시 체크포인트 기록 의무

### 성능 제약

- 폴더 비교: 10,000 파일 이하에서 5초 이내 완료
- UI 렌더링: 60fps 유지
- 메모리 사용: 200MB 이하

```
P0 = MVP 필수 (없으면 배포 불가)

P2 = 향상 기능 (이후 버전 가능)
```
