## UC-001 — 두 프로젝트 폴더 구조 비교

**사용자**: 개발자 A
**상황**: 두 버전의 프로젝트 폴더가 있고, 어떤 파일이 추가/삭제/변경되었는지 확인하고 싶다.

**시나리오 흐름**:
```
1. VS Code에서 Command Palette 열기 (Cmd+Shift+P)
2. "PlanEx3: Compare Folders" 명령 실행
3. 좌측 폴더 선택 (기준 폴더)
4. 우측 폴더 선택 (비교 대상 폴더)
5. 비교 결과가 트리 패널에 표시됨
6. 각 파일/폴더에 상태 아이콘 표시:
   - 🟢 ADDED: 우측에만 있음
   - 🔴 DELETED: 좌측에만 있음
   - 🟡 MODIFIED: 양쪽에 있으나 내용 다름
   - ⚪ SAME: 양쪽이 동일
7. 개발자 A는 변경 내역을 전체적으로 파악함
```

**성공 기준**:
- 5초 이내에 비교 결과 표시
- 모든 파일 상태가 정확하게 표시됨
- 중첩 디렉토리 구조 지원

## UC-002 — 배포 폴더와 소스 폴더 동기화

**사용자**: 개발자 B
**상황**: 소스 폴더의 최신 변경을 배포 폴더에 반영하고 싶다.

**시나리오 흐름**:
```
1. 폴더 비교 실행 (소스 폴더 vs 배포 폴더)
2. MODIFIED 또는 ADDED 상태 파일 목록 확인
3. 복사할 파일 체크박스로 선택
4. "우측으로 복사" 액션 실행
5. 확인 다이얼로그에서 복사 대상 목록 확인
6. 확인 클릭 → 파일 복사 실행
7. 복사 완료 후 상태가 SAME 으로 업데이트됨
```

**성공 기준**:
- 다중 파일 일괄 선택 및 복사 가능
- 복사 전 확인 단계 존재
- 복사 후 즉시 상태 업데이트

## UC-003 — 백업 폴더에서 특정 파일 복원

**사용자**: 개발자 C
**상황**: 실수로 파일을 삭제했고, 백업 폴더에서 특정 파일만 복원하고 싶다.

**시나리오 흐름**:
```
1. 폴더 비교 실행 (현재 폴더 vs 백업 폴더)
2. 현재 폴더에 없는 파일 = DELETED 상태로 표시
3. 복원할 파일을 찾아 선택
4. "좌측으로 복사" 액션 실행
5. 복원 완료
```

**성공 기준**:
- 양방향 복사 지원 (좌→우, 우→좌)
- 부분 복원 가능 (폴더 전체가 아닌 파일 단위)

## UC-004 — 두 텍스트 파일 내용 비교

**사용자**: 개발자 D
**상황**: 비교 트리에서 MODIFIED 파일을 클릭해 내용 차이를 확인하고 싶다.

**시나리오 흐름**:
```
1. 폴더 비교 뷰에서 MODIFIED 상태 파일 클릭
2. diff 뷰가 열림 (좌: 기준 파일, 우: 비교 파일)
3. 변경된 줄이 색상으로 강조 표시
   - 초록: 추가된 줄
   - 빨강: 삭제된 줄
   - 노랑: 변경된 줄
4. 개발자 D는 변경 내용을 라인 단위로 확인
```

**성공 기준**:
- VS Code 내장 diff 뷰 활용 또는 동등한 UI 제공
- 줄 단위 강조 표시

## UC-005 — 파일 변경 내용 선택적 병합

**사용자**: 개발자 E
**상황**: 두 파일의 변경 내용 중 일부만 적용하고 싶다.

**시나리오 흐름**:
```
1. diff 뷰에서 병합하고 싶은 변경 블록 선택
2. "Apply Hunk" 또는 "Accept Change" 액션 실행
3. 충돌 발생 시 충돌 마커 표시
4. 사용자가 직접 충돌 해결 (좌측/우측/양쪽 선택)
5. 결과 저장
```

**성공 기준**:
- Hunk 단위 선택적 적용 가능
- 충돌 마커를 통한 수동 해결 가능

## UC-006 — 특정 확장자만 필터링해서 비교

**사용자**: 개발자 F
**상황**: 대용량 폴더에서 `.ts` 파일만 비교하고 싶다.

**시나리오 흐름**:
```
1. 폴더 비교 뷰에서 필터 아이콘 클릭
2. "*.ts" 패턴 입력
3. 비교 결과가 .ts 파일만 표시되도록 필터링
4. 나머지 파일은 트리에서 숨김 또는 회색 처리
```

**성공 기준**:
- glob 패턴 기반 필터링
- 필터 상태 표시 및 해제 가능
