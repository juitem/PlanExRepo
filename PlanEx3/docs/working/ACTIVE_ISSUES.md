# ACTIVE_ISSUES — 활성 이슈 추적

> **갱신 규칙**: 이슈 발견 즉시 추가, 해결 시 상태 변경 (삭제하지 말 것 — 이력 보존)

---

## 이슈 상태 정의

```
OPEN     : 미해결, 작업 차단 없음
BLOCKING : 미해결, 현재 작업 차단 중
RESOLVED : 해결됨
DEFERRED : 나중으로 미룸 (사유 기록 필수)
WONTFIX  : 해결하지 않기로 결정 (ADR 참조 기록 필수)
```

---

## 현재 활성 이슈

```
현재 등록된 이슈 없음.
```

---

## 이슈 등록 템플릿

새 이슈 등록 시 아래 형식을 복사해서 추가:

```markdown
### ISSUE-XXXX — [제목]

| 항목 | 내용 |
|------|------|
| ID | ISSUE-XXXX |
| 상태 | OPEN / BLOCKING / RESOLVED / DEFERRED / WONTFIX |
| 발견일 | YYYY-MM-DD |
| 발견 에이전트 | 에이전트 역할명 |
| 관련 Job/Phase/Step | JOB-XXX / Phase X / Step X |
| 우선순위 | HIGH / MEDIUM / LOW |

**설명**
[이슈 상세 설명]

**영향 범위**
[어떤 작업에 영향을 주는가]

**임시 해결책 (있을 경우)**
[임시 우회 방법]

**해결 방법**
[해결 방법 또는 해결 계획]

**해결일 / 해결 에이전트**
[YYYY-MM-DD / 에이전트 역할명]
```
