# HYPOTHESES — 작업 가설 및 임시 판단

> **목적**: 에이전트가 작업 중 세운 가설, 임시 판단, 검증이 필요한 전제를 기록한다.
> **갱신 규칙**: 작업 시작 시 관련 가설 읽기, 종료 시 검증 결과 반영

---

## 가설 상태 정의

```
PROPOSED  : 제안된 가설, 미검증
VALIDATED : 검증됨 (근거 기록 필수)
REFUTED   : 반증됨 (반증 근거 기록 필수)
PROMOTED  : stable/ 또는 decisions/ 로 승격됨
```

---

## 활성 가설

```
현재 등록된 가설 없음.
```

---

## 가설 등록 템플릿

```markdown
### HYP-XXXX — [가설 제목]

| 항목 | 내용 |
|------|------|
| ID | HYP-XXXX |
| 상태 | PROPOSED / VALIDATED / REFUTED / PROMOTED |
| 생성일 | YYYY-MM-DD |
| 생성 에이전트 | 에이전트 역할명 |
| 관련 Job/Phase | JOB-XXX / Phase X |
| 검증 방법 | 어떻게 검증할 것인가 |

**가설 내용**
[가설 상세 설명]

**근거 (제안 시)**
[이 가설을 세운 근거]

**검증 결과**
[검증 후 결과 기록]

**승격 조건**
[어떤 조건이면 stable/ 로 승격할 것인가]
```
